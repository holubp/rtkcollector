# RtkCollector 1.0-RC1

Release date: 2026-07-16

This is a source-only release candidate. APK and AAB artifacts are intentionally
deferred until they have been built and signed on Windows with Android Studio,
checked for both supported ABIs and exercised on physical Android hardware.

## Highlights

- Byte-exact USB receiver recording remains authoritative and isolated from UI,
  parser, NTRIP, satellite-monitor and RTKLIB failures.
- Recording shutdown now confirms capture, correction, upload, advisory and
  RTKLIB workers are quiescent before session files are finalized.
- Session archive, restore, delete and temporary-share paths use active-session
  guards, bounded archive validation and rollback-safe file replacement.
- Settings imports validate the complete profile graph, sanitize unavailable SAF
  grants and use rollback-aware profile and secret-store transactions.
- NTRIP correction download and v1/v2 source upload include role-specific
  protocol handling, monitoring and configurable upload safety controls.
- UM980 and u-blox monitoring, temporary/fixed-base workflows, RTKLIB processing
  and satellite frequency/constellation views are included in this candidate.

## Important Limitations

- RtkCollector currently targets external USB GNSS receivers; it is not a phone's
  internal-GNSS application.
- UM980 and u-blox M8T are the primary tested receiver families. M8P and F9P may
  work through compatible u-blox modes, but precise positioning has not been
  validated by the author on those devices.
- NTRIP profiles currently permit cleartext TCP. Credentials, GGA positions and
  correction/source data are not encrypted unless the selected external service
  provides a separately secured transport.
- This remains release-candidate software. Validate recorded raw data and derived
  positions before operational or safety-relevant use.
- Google Play publication still requires a signed AAB, release-bundle native
  library verification, Android permission/foreground-service checks and field
  smoke tests described in `docs/play-publication.md`.
